This header is a repackaged and modified version of the \href{https://github.com/littlstar/b64.c}{\texttt{ b64.\+c}} library by Joseph Werle. The most significant change is that there is no dynamic memory allocation. Instead, there are functions that compute the size of encoded/decoded buffers in advance. Other changes are mostly cosmetic and are intended to make the code easier to understand.

\href{https://en.wikipedia.org/wiki/Base64}{\texttt{ Base64}} is a means of encoding binary data as plain ASCII. Each Base64 character represents log2(64) = 6 bits, meaning the encoded bytes occupy more memory that the original. This encoding is useful in circumstances where data needs to be stored or transmitted, but where a binary format is not possible nor desired. Applications include embedding binary data in JSON/\+XML, as well as representing cryptographic certificates and signatures in plain text.\hypertarget{pico__b64_8h_autotoc_md4}{}\doxysubsubsection{Usage\+:}\label{pico__b64_8h_autotoc_md4}
