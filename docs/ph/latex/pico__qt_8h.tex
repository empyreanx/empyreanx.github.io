This library builds and manages \href{https://en.wikipedia.org/wiki/Quadtree}{\texttt{ Quadtrees}}.

A quadtree is a data structure that can be used to perform efficient spatial queries. Items (values + bounds) are inserted into the tree. During this process, space in a quadtree is subdivided to make subsequent retrieval fast. Queries return values for all items that are contained within or overlap the search area.

Currently, values are numeric. If uintptr\+\_\+t is used they can also store a pointer. An integer value could represent an entity ID, an array index, a key for a hashtable etc...\hypertarget{pico__qt_8h_autotoc_md49}{}\doxysubsubsection{Depth\+:}\label{pico__qt_8h_autotoc_md49}
There is a tradeoff between space and time complexity. Lower depth values have smaller space requirements, but higher search times. Indeed, a tree with a depth of zero reduces to a linear bounds check. Higher values speed up searches, but at the cost of increased space. There are, however, diminishing returns with regard to increasing the max depth too high. Eventually all of the additional space is wasted with no benefit to performance.\hypertarget{pico__qt_8h_autotoc_md50}{}\doxysubsubsection{Usage\+:}\label{pico__qt_8h_autotoc_md50}
To use this library in your project, add the following

\begin{quote}
\#define PICO\+\_\+\+QT\+\_\+\+IMPLEMENTATION \#include \char`\"{}pico\+\_\+qt.\+h\char`\"{} \end{quote}
to a source file (once), then simply include the header normally.

Removes all items in the tree. 

This function preserves the internal structure of the tree making it much faster than {\ttfamily qt\+\_\+reset}. Reinserting values is probably faster too, however, repeated calls to this function may result in fragmentation of the tree. The function {\ttfamily qt\+\_\+clean} can repair this fragmentation, however, it is expensive.

Resets the tree and reinserts all items. 

This function can repair fragmentation resulting from repeated use of {\ttfamily qt\+\_\+remove} or {\ttfamily qt\+\_\+clear}. This is an expensive operation since it must extract all of the items from the tree, remove all of the nodes, and then reinsert all of the items.
