Features currently supported by the engine:

Foundational features. Most of the work involved contributions to pico_headers (ECS, renderer, math, collision detection/spatial paritioning, logging, unit testing), so what is listed here is only the tip of the iceberg.

  • Graphics (sprites/animations) using custom 2D renderer (with batching draw calls for performance)
  • Music and basic 2D positional audio
  • Basic 2D physics (linear and rotational kinematics, but only linear dynamics). This physics engine can be swapped out for a more powerful one if needed.
  • High-performance, multi-threaded architecture
  • ECS (Entity-Component-System) framework for rapid development of game logic
  • Tool support: PhysicsEditor, TexturePacker, and Tiled editor (orthogonal tile map loading and rendering)
  • Platform support via SDL for PC (Window, Linux, MacOS), mobile (Android, iOS), and possibly consoles (PS5, Xbox, Switch, and others...)

Planned features for alpha release (est. 2.5 months):

Essential features to be implemented before breaking ground on a full-fleged game (can happen in parallel to game design).

  • Write pico_ecs version 3.0 and integrate into existing codebase
  • Physics body/shapes serialization/deserialization (partially implemented)
  • Particle systems
  • Font handling/text rendering
  • Rendering context support for Metal and D3D
  • Indexed rendering
  • Basic AI support (finite state machines)
  • Raycasting (ECS component/system?)
  • HUD rendering/update
  • Menu system (microui for now)
  • Screen transitions/effects

Planned features for beta release (est. 3+ months):

These features will be implemented in parallel with game development. Some of the work will involve contributions to pico_headers.

  • Isometric tile map loading and rendering
  • Contact points/rotational dynamics
  • Dynamic tree for broadphase collision detection
  • More realistic positional audio
  • Networking support (HTTPS/UDP)/headless mode
  • More AI algorithms (movement, decision making, pathfinding, strategy and tactics)
  • Async resource loading
  • UTF-8/localization (i18n)
